## 贡献方式

### 报告 Bug

1. 搜索现有的 Issue 避免重复
2. 提供复现步骤和日志
3. 说明预期行为和实际行为

### 提出功能建议

1. 描述你的使用场景
2. 解释为什么需要这个功能
3. 提供可能的实现方案

### 提交代码

1. Fork 本仓库
2. 创建功能分支
3. 遵循代码规范
4. 提交 Pull Request

## 代码规范

### C++ 规范

- 遵循 Google C++ Style Guide
- 使用 C++17 或更高标准
- 添加必要的注释

### Qt 规范

- 使用 Qt 的信号槽机制
- 遵循 Qt 的命名约定
- 注意跨平台兼容性
