文件追踪是 Andrea Novel Helper 的核心功能之一，用于管理小说项目中的文档。

## 什么是文件追踪？

文件追踪系统通过为每个文档分配唯一的 UUID（通用唯一标识符），实现：
- 跨文件引用和关联
- 稳定的文件标识（即使文件移动或重命名）
- 项目结构的持久化存储

## 文件格式要求

| 格式 | 支持状态 |
|------|----------|
| `.md` | ✅ 完全支持 |
| `.txt` | ✅ 支持 |
| 其他格式 | ❌ 不支持 |

## 排除规则

以下文件不会被追踪：
- `novel-helper` 目录下的文件
- 符合 `.wcignore` 规则的文件

## 常见用途

### 角色引用

文件追踪使得角色高亮和自动补全功能能够跨文件工作。

### 关系图绑定

在关系图编辑器中，可以将关系绑定到特定的文档文件。

### 时间线关联

时间线编辑器支持将事件与具体章节文件关联。

## 故障排除

### 文件未被追踪？

1. 检查文件格式是否为 `.md` 或 `.txt`
2. 确认文件不在排除目录中
3. 尝试手动刷新追踪状态

### UUID 冲突？

如果遇到 UUID 相关问题，可以使用数据库管理命令重置。
